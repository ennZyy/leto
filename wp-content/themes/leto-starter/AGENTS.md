# AGENTS.md

## Окружение

- Разработка ведётся в директории `@wp-content/themes/leto-starter\`.
- Разработка ведётся локально (OSPanel).
- Проект доступен по адресу: http://leto/
- PHP 8.1, чистый WordPress 7.0.1, префикс таблиц БД: `wp_`.

## Стек и зависимости

- CMS: WordPress 7.0.1 (чистая установка).
- Язык: PHP 8.1.
- Плагины: ACF (бесплатная версия, не PRO), Contact Form 7.
- Архитектура темы: классические PHP-шаблоны
  (`header.php`, `footer.php`, `page.php`, `single.php` и т.д.) + `functions.php`
  и отдельные классы/модули.

## Соглашения по коду

- Стиль кода: PSR-12.
- Язык общения и заметок: русский.
- Комментарии в коде не добавлять, если явно не попросят.
- Не делать `git commit` без прямой просьбы пользователя.
- Для функций и классов темы использовать префикс/нейминг темы `leto-starter`,
  для таблиц БД — префикс `wp_`.

## Обновление документации

- При значимых изменениях/доработках темы своевременно обновлять MD-файлы
  темы, чтобы они оставались актуальным источником контекста для ИИ-агентов.
- Описание дизайн-системы, структуры шаблонов и секций ведётся в
  `THEME.md` — при добавлении/изменении шаблона, секции (`template-parts`),
  цвета, шрифта или меню нужно отразить это в `THEME.md`.
- При смене стека, окружения или процесса обновлять `AGENTS.md`.

## Процесс работы и тестирование

- Перед изменениями изучать существующие шаблоны и классы темы,
  соблюдать принятые соглашения и стиль.
- После выполнения задачи проверять результат через Playwright:
  открывать страницу по адресу `http://leto/` и снимать скриншоты
  для визуальной проверки вёрстки и отсутствия ошибок в консоли.
- Запуск Playwright: локальный `npx playwright` скрипт, настроенный в проекте.
